## Intent
Throttle brute-force login attempts by adding per-IP and per-account rate limiting to `POST /login`.

## Key review outcomes
- Reviewer (@aravind) required a **dual limit** — per-IP *and* per-account — not just per-IP. (thread)
- Reviewer (@priya) held the PR until the migration added a reversible `down` step (`repo-dna: reversible-migrations`).
- Input validation on the new middleware config was requested and added (`review-memory: validate-at-boundary`).

## Lessons / gotchas
- `src/middleware/rate-limit.ts` sits in the hot path of `svc:auth`; changes here are **high-risk** and should pull extra graph neighbors + a security pass.
- Counter storage is in-memory in v1 — a follow-up to move it to Redis is tracked separately.
